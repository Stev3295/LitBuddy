# LitBuddy AI - 文献智能管家（DOI去重·中文增强·详细版）

## 描述
批量处理 PDF 论文，精确提取标题、作者、摘要（中文化）、关键词，智能概括研究方法与核心结论（各 150–200 字），生成中文标签。通过 DOI 识别重复文献，仅保留最新版本。**只做两件事**：
1. 在论文所在文件夹生成汇总报告 `literature_summary.md`
2. 更新固定位置的 JSON 数据库 `C:\Users\abc\.openclaw\workspace\literature_db.json`

**严禁创建任何子文件夹，严禁移动原始 PDF 文件。**

## 触发方式
- “批量整理这个文件夹里的论文：C:\papers”

## 执行步骤（严格按顺序）

### 步骤 0：检查并报告文件夹状态
列出用户指定文件夹的内容。如果发现任何子文件夹，**立即输出警告**并要求用户手动删除。

### 步骤 1：获取 PDF 文件列表
扫描文件夹，收集所有 `.pdf` 文件的绝对路径。

### 步骤 2：逐个处理 PDF（仅内存操作）
对每个 PDF 执行以下子步骤：

#### 2.1 文本提取
- 优先 `pdfplumber`（`encoding='utf-8'`）
- 失败则 `PyMuPDF`（`fitz`）
- 仍失败则标记“解析失败”

#### 2.2 提取 DOI（关键去重依据）
从 PDF 元数据或文档前几页中查找匹配模式：
- `10.\d{4,9}/[-._;()/:A-Z0-9]+`（大小写不敏感）
- 也支持常见变体如 `doi:`、`DOI:` 后跟的字符串。
- 如果未找到 DOI，则使用 `filePath` 作为备用标识（但优先 DOI）。

#### 2.3 提取标题（精确规则）
按优先级：
1. 从 PDF 元数据 `title` 获取，排除期刊特征词。
2. 从第一页顶部提取非期刊名、非页码的完整句子。
3. 排除包含 `Journal`、`Vol`、`No`、`Page`、`DOI`、`通讯作者` 等词语的行。
4. 对于混合文本（如“第 X 卷 第 X 期 作者:标题”），只保留冒号后的标题部分。

#### 2.4 提取作者
- 查找“作者”、“Authors”下方行，提取至句号或换行。

#### 2.5 提取摘要（强制中文化）
- 查找 “Abstract”、“摘要” 段落。
- **如果原文有中文摘要**：直接使用（限制 200 字内）。
- **如果只有英文摘要**：将英文摘要翻译成中文（保持学术语气），长度控制在 200 字左右。翻译时注意专业术语准确。
- 翻译方法：AI 自行翻译，不得保留原文。

#### 2.6 提取关键词（中文化）
- 查找 “Keywords”、“关键词” 后的词列表。
- 若为英文关键词，翻译成中文（如 `catalyst` → `催化剂`）。
- 输出为中文逗号分隔字符串。

#### 2.7 生成主题标签（中文化 + 外文标记）
- 基于标题、摘要、关键词生成 3-5 个**中文标签**（如 “二氧化碳转化”、“金属有机框架”）。
- **如果原文主体语言为英文**：额外添加一个固定标签 `[英文]`。
- 标签示例：`催化剂, 碳酸二甲酯, 铈基材料, [英文]`（若适用）。

#### 2.8 提取研究方法（150–200 字，详细但清晰）
**目标**：从 “Methodology”、“Methods”、“实验部分”、“材料与方法” 等章节提取核心实验设计，长度 150–200 字。

步骤：
- 定位上述章节，提取 2-4 个完整句子。
- **必须清洗**：删除引用编号（如[16]）、乱码字符、多余换行、异常空格。
- 重组句子，确保连贯。保留关键信息：催化剂制备、反应条件（温度、压力、时间）、表征手段、催化性能指标（转化率、选择性）。
- 如果原文过长，可适当压缩但不得少于 120 字。
- 若找不到方法章节，从引言末尾或结果讨论开头提取相关描述。
- 输出示例（约 180 字）：“采用浸渍法制备了不同 CeO2 负载量的 ZIF-67 复合催化剂。XRD 和 TEM 表明 CeO2 纳米颗粒均匀分散于 ZIF-67 表面。活性评价在高压反应釜中进行，反应条件：CO2 压力 2.6 MPa，温度 150°C，反应时间 6 小时。结果表明，10% CeO2/ZIF-67 的 DMC 收率达到 55.4%，优于纯 ZIF-67 和 CeO2。”

#### 2.9 提取基本结论（150–200 字，核心发现）
**目标**：从 “Conclusion”、“Results”、“结论” 等章节提取核心结论，长度 150–200 字。

步骤：
- 定位上述章节，提取 2-4 个核心结论句子。
- 清洗乱码、引用标记，重构语句。
- 突出关键数据（如收率、选择性、稳定性）。
- 若原文结论较短，可适当扩展解释，但保持忠实。
- 输出示例（约 170 字）：“研究发现，CeO2 的引入显著提高了 ZIF-67 对 CO2 和甲醇直接合成碳酸二甲酯（DMC）的催化活性。在最优条件下（150°C, 2.6 MPa CO2），DMC 收率达到 55.4%，选择性高于 99%。催化剂重复使用 5 次后活性无明显下降。DFT 计算表明 CeO2 与 ZIF-67 的界面协同效应促进了 CO2 的活化。”

#### 2.10 存储到内存
将以上字段存入 `papers_data` 列表，每条记录结构：
```json
{
  "doi": "10.xxxx/xxxxx",
  "title": "标题",
  "authors": "作者",
  "abstract": "中文摘要（200字内）",
  "keywords": "中文关键词",
  "tags": ["中文标签1", "中文标签2", "[英文]（如需）"],
  "methods": "研究方法（150-200字）",
  "conclusion": "基本结论（150-200字）",
  "filePath": "PDF绝对路径",
  "dateAdded": "ISO时间戳"
}
